EFCore相关的从5.0.4 升级到 5.0.6

 

AutoFac从6.1.0 升级到6.2.0

EPPlus 从 5.5.5 升级到 6.6.4

JWT  从8.1.3 升级到 8.2.2

protobuf-net 从 3.0.73 升级到 3.0.101

Z.EntityFramework.Plus.EFCore 从5.1.23 升级到5.1.39  

 

 

EFCore.BulkExtensions从3.3.5 升级到 5.2.2   （好像要收费？？）

Pomelo.EntityFrameworkCore.MySql 从 5.0.0-alpha.2版本 升级到 5.0.0 版本   ，注册代码如下：

services.AddDbContext<CoreFrameDBContext>(option => option.UseMySql(ConfigHelp.GetString("ConnectionStrings:MySQLStr"), Microsoft.EntityFrameworkCore.ServerVersion.Parse("5.7.28-mysql")));

Zack.EFCore.Batch.MSSQL 从1.2.4 升级到1.3.0

Zack.EFCore.Batch.MySQL.Pomelo  从1.2.4 升级到1.3.0   在上下文实体中间，简单的判断了一下：

  protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
        {
            optionsBuilder.UseLoggerFactory(LoggerFactory.Create(build =>
            {
                build.AddDebug();
            }));
      
            if (ConfigHelp.GetString("DBType") == "MySQL")
            {
                optionsBuilder.UseBatchEF_MySQLPomelo();//MySQL 专用
            }
            else
            {
                optionsBuilder.UseBatchEF_MSSQL();//SQLServer 专用
            }
        }
